Что надо строить? 
Гистограмму (как на рис. 3 или 4 из статьи в пдф, стр. 14) по данным из файла statistics, или как-то иначе. У них true product value, у нас - поле userEstimate.


Описание данных в файлах.

==providers.txt==
providerID, serviceQuality (только для данного эксперимента)

общаЯ: providerID, <параметр1, параметр 2>

providerID - номер (ID) провайдера
serviceQuality - в данном эксперименте провайдер может предоставлять сервис только фиксированного качества, которое задается в начале константой.

==tasks.txt==
userID, arrivalTime

userID - номер (ID) пользователя, от которого пришла заявка.
arrivalTime - время прихода задания.

==users.txt==
userID, neededServiceQuality 

userID - номер (ID) пользователя
neededServiceQuality - минимальное значение качества сервиса, требуемое пользователем.

==results.txt==
пользовательская оценка предоставленного ему сервиса - усредненная за н итераций (каждый раз одинаковое количество заданий). 
(То же, что в старых результатах в файле average.txt)

==statistics.txt==
arrivalTime, userID, providerID, serviceQuality, userEstimate

arrivalTime - время прихода задания
userID - номер (ID) пользователя, от которого пришло задание
providerID - номер (ID) провайдера, которого выбрала система для обслуживания запроса
serviceQuality - качество сервиса*, предоставленного этим провайдером
userEstimate - пользовательская оценка предоставленного сервиса

*качество сервиса = один из параметров, по которым производится оценка сервиса.
